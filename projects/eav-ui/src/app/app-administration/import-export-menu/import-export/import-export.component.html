Import Export

2sxc provides you with many import-export features for all kinds of scenarios.

Export/Import of Data Lists, Schemas, Views and Queries

Before you continue here, you should know that some import/export features are directly available in other parts of the admin UI.

  1. Lists of Data can be exported / imported as XML (for editing in Excel) in the [data] section.
  2. Content-Types can be exported and imported as JSON in the [data] section.
  3. Views can be exported and imported as JSON (with the template files) in the views section.
  4. Queries can be exported and imported as JSON in the queries section.

App Export / Import

Apps are exported as a ZIP file which can be imported into another site as needed. Export happens here, import is done in the Manage Apps section.

Data Bundles (beta)

Data Bundles are a new feature which allows you to export and import data from one app to another. This is very useful when you want to sync only parts of the data/schema from one environment to another. Bundles allow you to define what is in the bundle, and repeat the sync many times as the system changes.

Export / Import Parts (old)

This feature is the old way of exporting and importing parts of an app. It's still available, but we recommend using the new Data Bundles feature instead. The parts allow you to select some data and export it as XML, then re-import it elsewhere.

App State Backup and Git Sync

This feature is for advanced users who want to backup the entire App State. The App State contains all the data, schema, views, queries and more. This allows you to:

  1. Do backups, and reset your App to what was in that backup.
  2. When combined with Git, you can also reset the App to previous states.
  3. When combined with Git, you can also use it to transfer the complete App State from one site to another.
